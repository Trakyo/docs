## Base URL

```
https://api.trakyo.io
```

## Authentication

All endpoints require a Bearer token in the `Authorization` header:

```
Authorization: Bearer tky_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

See [Authentication](/authentication) for details on obtaining and managing API keys.

## Response format

Successful responses return JSON with endpoint-specific fields. List endpoints return a Stripe-style envelope: `{ "object": "list", "data": [...], "has_more": boolean, "url": "/v1/..." }`.

Error responses use a nested `error` envelope — see [Errors](/concepts/errors) for the full shape.

Every response (success or error) carries an `X-Request-Id` header (`req_<hex>`). Include it when reporting issues — error bodies expose the same value as `request_id`.

## CORS

The API allows cross-origin requests from any origin (`Access-Control-Allow-Origin: *`). Rate-limit headers and `X-Request-Id` are exposed to browsers via `Access-Control-Expose-Headers`.

## Idempotency

`POST` requests accept an optional `Idempotency-Key` header (up to 255 characters). If you retry a request with the same key, the API returns the original response without creating a duplicate resource.

## OpenAPI specification

The API self-serves its OpenAPI 3.1 spec at:

```
GET https://api.trakyo.io/openapi.json
```

Use this to generate client SDKs or import into API tools like Postman.

## Health check

```
GET https://api.trakyo.io/health
```

Returns `{"status":"ok"}` when the API is operational. No authentication required.

## Link identifier

Links are identified by a composite key of `domainId` (UUID) + `key` (slug). Use the sentinel domain ID `00000000-0000-0000-0000-000000000000` for links on the default `trakyo.link` domain.
