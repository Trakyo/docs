---

The Trakyo API uses [RFC 9457 (Problem Details for HTTP APIs)](https://www.rfc-editor.org/rfc/rfc9457.html) for all error responses. This is the same standard used by Cloudflare, Stripe, and other major APIs.

## Error format

Every error response uses the `application/problem+json` content type and includes these fields:

```json
{
  "type": "https://trakyo.io/docs/errors/not_found",
  "title": "Not found",
  "status": 404,
  "detail": "No link found with key 'abc123' on domain '00000000-0000-0000-0000-000000000000' for this account.",
  "instance": "/api/v1/links/00000000-0000-0000-0000-000000000000/abc123"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `type` | string | URI identifying the error. Links to this documentation. |
| `title` | string | Short human-readable summary. |
| `status` | integer | HTTP status code. |
| `detail` | string | Detailed explanation of what went wrong. |
| `instance` | string | The request path that caused the error (when applicable). |

## Error codes

The `type` field in every error is a URL that links to a detailed reference page for that error.

| Status | Code | Description |
|--------|------|-------------|
| 400 | [bad_request](/errors/bad_request) | Invalid parameters or request body |
| 401 | [unauthorized](/errors/unauthorized) | Missing or invalid API key |
| 401 | [key_expired](/errors/key_expired) | API key has expired |
| 403 | [forbidden](/errors/forbidden) | API key doesn't have access to this resource |
| 404 | [not_found](/errors/not_found) | Resource doesn't exist or doesn't belong to your account |
| 409 | [conflict](/errors/conflict) | Resource already exists (e.g., slug taken) |
| 429 | [rate_limited](/errors/rate_limited) | Too many requests |
| 500 | [internal_error](/errors/internal_error) | Something went wrong on our end |
